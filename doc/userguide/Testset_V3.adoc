=== 1. Einführung und Gültigkeit

{empty} +

In den nachfolgenden Kapiteln finden Sie sämtliche zusätzlichen Informationen zu den bereitgestellten Testfällen sowie zur Funktionalität des *Testset Version 3* für den TI-Messenger.

Diese Dokumentation ergänzt die bereits vorhandene Datei link:Testsuite.adoc[Testsuite.adoc], in der die grundlegenden Funktionen der Testsuite ausführlich beschrieben ist.

Hier werden ausschließlich zusätzliche Punkte behandelt, die speziell das Testset in der Version 3 betreffen.
Für weiterführende Informationen verweisen wir auf link:Testsuite.adoc[Testsuite.adoc].

Alle wichtigen Informationen für die Hersteller sind wie folgt gekennzeichnet.

=== 2. Tags und Testausführung des Testsets zur Zulassung

[.id=kapitel-Rollen]

image::Tag.png[float="left"]

==== 2.1 Tags

*Tags zur Steuerung des Testsets zur Zulassung:*

[cols='1,3']
|===
| *Tag Name*        | *Beschreibung*
| @ZulV3:Pro        | Testfälle relevant für eine TI-M Pro Zulassung
| @ZulV3:ProKK      | Testfälle relevant für eine TI-M Pro Zulassung für Krankenkassen
| @ZulV3:ePA        | Testfälle relevant für eine TI-M ePA Zulassung
| @DeltaV3:Pro      | Neue/angepasste TI-M Pro Testfälle, die das Delta zur vorigen Testsetversion abbilden
| @DeltaV3:ProKK    | Neue/angepasste TI-M Pro für Krankenkassen Testfälle, die das Delta zur vorigen Testsetversion abbilden
| @DeltaV3:ePA      | Neue/angepasste TI-M ePA Testfälle, die das Delta zur vorigen Testsetversion abbilden
|===

*Client Tags in den Testfallfiltern:*

Die folgenden Filter wurden für das TI-Messenger Testset V3 angepasst und dienen zur Identifizierung von Clients, die mit dem Testset V3 kompatibel/nutzbar sind.

[cols='1,3']
|===
| *Filter Name*        | *Beschreibung*
| proClientV3          | Filter für einen Pro-Client zur Nutzung des Testsets V3 bzw zur Generierung der unterschiedlichen Test Examples
| epaClientV3          | Filter für einen ePA-Client zur Nutzung des Testsets V3 bzw zur Generierung der unterschiedlichen Test Examples
|===

Grundlage für die oben aufgeführten Tags sind die Werte aus der Infoschnittstelle, beschrieben in der TiMessengerTestTreiber.yaml (*src/main/resources/api/TiMessengerTestTreiber.yaml*, operationId: getInfo).

==== 2.2 Testausführung verschiedener Zulassungstestsets

{empty} +

[[img_folderStructure2]]
.Abbildung{counter:my-example-number: 1}: Ordnerstruktur der TI-Messenger Testsuite
image:Folder_V3.png[float="center"]

Der "Basis" Folder ist für alle Zulassungen angelegt und enthält Tests zu Anwendungsfällen und Anforderungen, die sich aus der TI-M Basis Spezifikation ergeben.

Aufbauend hierauf enthalten die Folder "Pro" und "ePA" weiterführende Tests zu den jeweiligen Spezifikationen, die ebenfalls entsprechende Anwendungsfälle und Anforderungen beinhalten.

Hieraus ergeben sich verschiedene Testsets, die über diverse Tags unterschieden werden können:

- *Zulassung TI-M Pro*

Auszuführen z.B. mit `mvn clean install -Dcucumber.filter.tags=@ZulV3:Pro`

- *Zulassung TI-M Pro für Krankenkassen ohne HBA Funktionalität*

Auszuführen z.B. mit `mvn clean install -Dcucumber.filter.tags=@ZulV3:ProKK`

- *Zulassung TI-M ePA*

Auszuführen z.B. mit `mvn clean install -Dcucumber.filter.tags=@ZulV3:ePA`

{empty} +

Für Tests zur Erfüllung eines Teils der Eingangskriterien (Herstellerinstanz A gegen Herstellerinstanz B) ergeben sich folgende Befehle, da hier die Tests, die eine Referenzimplementierungsinstanz zwingend voraussetzen, exkludiert werden müssen:

- *Zulassung TI-M Pro*

Auszuführen z.B. mit `mvn clean install -Dcucumber.filter.tags="@ZulV3:Pro and not @Ref:Mandatory"`

- *Zulassung TI-M Pro für Krankenkassen ohne HBA Funktionalität*

Auszuführen z.B. mit `mvn clean install -Dcucumber.filter.tags="@ZulV3:ProKK and not @Ref:Mandatory and not @Ctl:ProKKHBA"`

{empty} +
